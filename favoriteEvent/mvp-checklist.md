# 찜 정모 MVP – 개발 체크리스트

> 기능 티켓 = 사용자 기능 + 분석 가능성까지 포함

## 1. 정모 노출 처리 + 노출 이벤트 계측

### 목적

* 사용자가 특정 정모를 인식한 시점을 기록한다.
* 정모가 여러 방식으로 보여져도 하나의 이벤트로 관리한다.

### 기능 작업

- [x] 모임 화면에서 정모 리스트 렌더링
- [x] 정모 리스트에서 해당 셀 클릭 시, 모임 화면 내 해당 정모 위치로 포커싱

### 계측 작업

- [ ] `event_exposed` 이벤트 정의 및 전송
- [ ] 동일 사용자 기준으로, 동일 정모에 대해 최초 1회만 기록(화면 이동/스크롤로 재노출되더라도 중복 기록하지 않음)
- [ ] 동일 정모에 대해 중복 전송 방지

### 이벤트 조건

* 모임 화면 진입 시 정모 카드가 화면에 노출된 경우
* 정모 리스트에서 클릭 후, 해당 정모가 화면에 포커싱된 경우

**파라미터**

* `user_id`
* `event_id`
* `group_id`
* `source_category`
  * `event_context` (정모를 보고 들어온 경우)
  * `group_context` (모임 화면 내 정모를 본 경우)
* `exposure_method`
  * `list_visible`  (카드가 자연스럽게 화면에 노출된 경우)
  * `scroll_focus`  (해당 정모가 포커싱된 경우)

## 2. 정모 찜 / 해제 기능 + 찜 이벤트 계측

### 목적

* 사용자가 정모에 관심을 저장하거나 해제할 수 있도록 한다.
* 찜 행동을 이벤트 단위로 기록한다.

### 기능 작업

- [ ] 모임 - 정기 모임 - 정모 카드 하단에 '찜 버튼(아이콘)' 추가
- [ ] 모임 - 정기 모임 - 정모 카드 하단에 '공유 버튼(아이콘)' 수정
- [ ] 찜 / 해제 토글 동작 구현
- [ ] 동일 정모 중복 찜 방지

### 계측 작업

- [ ] 찜 시 `event_saved` 이벤트 전송
- [ ] 찜 해제 시 `event_unsaved` 이벤트 전송

**파라미터**

* `user_id`
* `event_id`
* `group_id`
* `timestamp`

## 3. 마이페이지에 ‘찜한 정모’ 버튼 추가

### 목적

* 사용자가 찜한 정모 리스트로 진입할 수 있게 한다.
* 마이페이지에서 찜 정모의 재확인을 유도한다.

### 기능 작업

- [ ] 마이페이지 상단 요약 영역에 ‘찜한 정모’ 버튼 추가
- [ ] 기존 ‘찜 모임 / 최근 본 모임 / 초대받은 모임’과 동일한 UI 패턴 사용
- [ ] 버튼 클릭 시 ‘찜한 정모 리스트’ 화면으로 이동
- [ ] 찜한 정모 개수 표시

### 계측 작업

- [ ] 마이페이지에서 버튼 클릭 시 이벤트 기록

```
event_name: saved_event_entry_clicked
```

**파라미터**

* `user_id`
* `entry_point`: `mypage`
* `timestamp`

## 4. 찜한 정모 리스트 화면 구성

### 목적

* 사용자가 찜한 정모를 한 화면에서 확인하고 다시 접근할 수 있게 한다.

### 기능 작업

- [ ] 찜한 정모 리스트 조회 API 연동
- [ ] 리스트 아이템은 아래 UI와 동일
  - [ ] 모임 - 정기모임 - 정모 카드(D-day 태그부터 버튼까지)
  - [ ] 가입한 모임이 아닌 경우 '참석' 텍스트 대신 '모임 가입하고 참석'
  - [ ] 정모가 끝난 경우, '참석' 텍스트 대신 '모임 둘러보기'
  - [ ] 정모가 끝난 경우, 버튼 보더, 텍스트 색상 진한 회색으로 변경
- [ ] 리스트 아이템 클릭 시 해당 정모가 포함된 **모임 화면으로 이동**
  - [ ] 정모가 진행중인 경우 해당 정모 위치로 포커싱
  - [ ] 정모가 끝난 경우 '정기모임' 섹션 포커싱

### 계측 작업

- [ ] 찜한 정모 리스트 진입 시 이벤트 기록
- [ ] 찜한 정모 리스트에서 정모 클릭 시, event_exposed 이벤트 재사용
  * `source_category`: `saved_event_list`
  * `exposure_method`: `scroll_focus`


```
event_name: saved_event_list_viewed
```

**파라미터**

* `user_id`
* `source`: `mypage`
* `timestamp`

## 5. 정모 종료 처리 + 참여/가입 이벤트 계측

### 목적

* 찜 이후 실제 행동(참여/가입)을 측정한다.

### 기능 작업

- [ ] 정모 종료 시 상태 업데이트
- [ ] 종료된 정모를 찜 목록에서 ‘종료’로 표시
- [ ] 찜 데이터는 유지 (삭제하지 않음)

### 계측 작업

- [ ] 정모 참여 확정 시 `event_joined` 이벤트 전송
- [ ] 모임 가입 완료 시 `group_joined` 이벤트 전송
- [ ] 동일 사용자 기준으로 이벤트 연결 가능하도록 전송

### 계측 작업

# 홈 개인화 섹션 노출 기획서 (찜 정모 기반)

본 문서는  
**찜 정모 데이터를 활용해 홈 화면에서 단 하나의 개인화 섹션만 노출**하기 위한  
우선순위 규칙 및 노출 기준을 정의한다.

---

## 1. 기획 목적

- 홈을 **탐색 공간이 아닌 결정 가속 공간**으로 유지한다.
- 찜 데이터를 전환 지표가 아닌 **노출 우선순위 신호**로 활용한다.
- 개인화 섹션 과노출로 인한 무시(배너화)를 방지한다.

---

## 2. 기본 원칙

1. 홈 개인화 섹션은 **0~1개만 노출**한다.
2. 조건이 여러 개 충족되더라도 **가장 강한 신호 1개만 선택**한다.
3. 조건이 충족되지 않으면 **아무 것도 노출하지 않는다.**

---

## 3. 핵심 개념 정의

### 찜한 정모
- 사용자가 `찜` 액션을 수행한 정모
- 상태: 예정 / 모집중 / 마감

### D-3 이내
- 현재 시점 기준 정모 시작일까지 3일 이하

### 마감된 찜 정모
- 사용자가 찜했으나 모집 마감 상태가 된 정모

---

## 4. 홈 개인화 섹션 우선순위 규칙

```pseudo
if (찜한 정모 중 D-3 이내 존재):
    show [곧 시작하는 찜한 정모]
else if (마감된 찜 정모 존재 && 같은 모임의 새 정모 존재):
    show [같은 모임의 새 정모]
else if (찜한 모임의 예정 정모 존재):
    show [찜한 모임의 다음 정모]
else:
    show nothing
```

---

## 5. 섹션별 상세 기획

### 5.1 곧 시작하는 찜한 정모 (최우선)

**노출 조건**

* 찜한 정모 중 시작일이 D-3 이내인 항목이 존재

**노출 목적**

* 즉시 행동 유도
* 결정 비용 최소화

**UI 형태**

* 리스트형 (최대 3개)
* 홈 상단 영역 고정

**CTA 규칙**

* 모임 가입 O → `참석`
* 모임 가입 X → `모임 가입하고 참석`

---

### 5.2 같은 모임의 새 정모 (마감 대체)

**노출 조건**

* 마감된 찜 정모가 존재
* 해당 정모를 만든 모임에서 새로운 정모가 생성됨

**노출 목적**

* 실패 경험을 다음 기회로 전환
* 신뢰 전이를 통한 판단 비용 감소

**UI 형태**

* 카드 1개
* 배너 또는 강조 카드

**노출 제한**

* 마감된 정모 1건당 1회 노출
* 동일 모임 반복 노출 금지

---

### 5.3 찜한 모임의 다음 정모

**노출 조건**

* 찜한 모임이 존재
* 해당 모임에 예정된 정모가 존재

**노출 목적**

* 관계 유지
* 부드러운 재참여 유도

**UI 형태**

* 카드 1개
* 홈 중단 또는 하단 영역

**CTA 규칙**

* `정모 보러가기`

---

## 6. 노출 제외 규칙

* 개인화 섹션을 **2개 이상 동시에 노출하지 않는다.**
* 모든 조건이 충족되지 않을 경우:

  * 기본 홈 콘텐츠 유지
  * 빈 섹션, 안내 문구, 대체 추천 없음

---

## 7. 비고

* 본 규칙은 서버 또는 클라이언트 단에서 결정 가능하나,
  **우선순위 판단 로직은 단일 책임으로 유지**한다.
* 추천 실패 시 침묵(no-op)을 기본값으로 한다.
* 개인화 설명 문구는 최소화하며, 노출 이유를 사용자에게 직접 설명하지 않는다.

---

## 8. 요약

* 홈 개인화는 “많이 보여주기”가 아니라
  **“가장 맞는 하나를 고르는 문제”**로 정의한다.
* 찜 정모는 관심 데이터이며,
  홈에서는 **가장 행동 가능성이 높은 순간에만 사용**한다.

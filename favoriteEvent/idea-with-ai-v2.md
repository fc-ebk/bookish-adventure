# 찜 정모 전략 아카이브

## 1. 문제 정의

- 정모 참석을 위해서는 모임 가입이 필요함
- 사용자는 정모에 관심이 있어도 즉시 모임 가입까지 이어지지 않음
- 기존 구조는 “지금 결정하지 않으면 이탈”로 이어지기 쉬움

핵심 문제:
> 관심은 있는데, 결정까지 가기엔 심리적/정보적 부담이 큼

---

## 2. 전략적 접근 요약

- 모임보다 앞선 최소 관심 단위를 **정모**로 설정
- 즉각적인 행동을 강요하지 않고, **관심을 저장하고 다시 돌아올 수 있는 구조**를 만든다
- 기능 → 루프 → 넛지를 단계적으로 쌓는다

---

## 3. 단계별 전략 개요

### 1차 (MVP): 찜 정모 기본 검증

#### 목표
- 정모 찜이 실제 참여/모임 가입으로 이어지는지 검증

#### 핵심 기능
- 정모 찜 / 해제
- 찜한 정모 리스트
  - 진행 예정 정모만 노출
  - 정렬: 오늘/내일, 이후 일정
  - 기존 정모 카드 압축 UI
- 알림
  - 마감된 정모에 자리 발생 시에만 발송

#### 데이터
- BigQuery 기반 행동 로그
  - 찜 / 해제
  - 참여 실행
  - 모임 가입 실행
  - 알림 유입 여부(source)

#### 의도
- “관심을 저장할 수 있으면, 사용자는 다시 돌아오는가?”

---

### 2차: 마감 이후 루프 확장

#### 문제
- 찜한 정모가 마감되면 흐름이 끊김

#### 전략
- 마감 이후 추천 우선순위
  1. 같은 모임의 다른 진행중/예정 정모
  2. 없을 경우에만 유사 정모
- 노출 위치
  - 찜 리스트 제외
  - 홈 화면 개인화 영역

#### 의도
- 실패 경험을 다음 기회로 전환
- 정모 단위 관심 → 모임 단위 관계로 확장

---

### 3차: 넛지 강화 (배너 전략)

#### 전제
- 찜/참여/가입 데이터가 충분히 누적됨
- 관심은 있으나 행동하지 않은 사용자 식별 가능

#### 전략
- 강한 넛지는 **관심이 검증된 사용자에게만**
- 조건 기반 노출
  - 반복 찜
  - 동일 모임 관심 누적
  - 찜 후 미참여 상태
- 위치
  - 홈 중단 개인화 영역

#### 넛지 성격
- 행동 강요가 아닌 맥락 상기
- “지금 가세요”가 아니라 “관심 가졌던 맥락”을 다시 보여줌

---

## 4. 데이터 전략 정리

### 기본 원칙
- GA 미사용, BigQuery 단일 소스
- 노출보다 **행동 로그 중심**

### 핵심 로그
- event_saved / event_unsaved
- event_joined
- group_joined
- source (notification / direct / home 등)

### 목적
- 1차: 전환 검증
- 2차: 루프 성과 확인
- 3차: 개인화 및 넛지 타게팅 기반

---

## 5. 확장 아이데이션 (자유 메모)

### 5.1 찜의 미세 상태 분리 (후순위)
- 찜 직후
- 오래된 찜
- 반복적으로 찜한 유형
→ UI 노출은 안 하되, 내부 점수로 활용 가능

---

### 5.2 찜 기반 소프트 랭킹
- 홈 정모 노출 시
  - “찜한 사용자 수”
  - “찜 → 참여 전환률”
를 랭킹 신호로 사용

---

### 5.3 찜 실패 경험의 재해석
- 마감 = 실패가 아니라 “인기 신호”
- “이 정모는 마감됐어요” 대신
  - “이 모임의 다른 정모는 어때요?”

---

### 5.4 찜 데이터의 장기 가치
- 찜은 참여하지 않아도 의미 있음
- “무엇을 좋아했는지”의 흔적
- 추천, 큐레이션, 알림 타게팅에 재사용 가능

---

### 5.5 알림 확장 가능성
- 1차: 자리 발생
- 이후:
  - 비슷한 정모 오픈
  - 관심 모임 신규 정모
- 단, 항상 조건 기반 + 제한적 노출

---

## 6. UX 관련 메모

- 찜은 ‘약한 약속’이어야 함
- 언제든 해제 가능해야 신뢰가 유지됨
- 리스트 UI는 탐색보다 결정에 최적화
- 새로운 카드 UI 최소화, 기존 패턴 재사용

---

## 7. 참고 레퍼런스 (개념적)

### 관심 저장 패턴
- Netflix: My List
- YouTube: Watch Later
- Airbnb: Saved Homes
- Meetup: Save Event

### 실패 후 루프 설계
- Airbnb: Sold out → Similar stays
- ClassPass: Full class → Same studio next class
- 공연 예매 플랫폼: 매진 → 같은 아티스트/공연 추천

### 넛지 활용
- Duolingo: streak 기반 배너
- Spotify: 반복 청취 기반 추천 배너
- Notion: 사용 패턴 기반 팁 노출

---

## 8. 한 줄 정리

찜 정모는 즉각적인 행동을 강요하지 않고 관심을 저장해 흐름을 이어주며,  
단계적으로 루프와 넛지를 얹어 참여·모임 가입·개인화를 확장하는 구조다.

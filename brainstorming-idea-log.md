# 브레인스토밍 아이디어 로그

## 목적 요약
- 정모 참여가 외부 플랫폼(카카오톡 등)으로 이탈되지 않고 앱 내부에서 발생하도록 유도
- 신규 모임·신규 사용자에 불리하지 않은 구조 설계
- 정모를 통한 유입·참여·개인화의 확장 가능성 검토

---

## 1. 정모/모임 참여 인원 기반 Advantage / Disadvantage 설계

### 아이디어
- 정모 참석자가 **일정 기준 이상**일 경우 advantage 제공
- 기준 미만일 경우 상대적 disadvantage 발생
- 목적: 외부 채널로 이동하지 않고 앱 내 정모 참여 유도

### 핵심 고려사항
- 기준은 절대값(예: 3명)이 아닌 **정원 대비 비율**이 더 합리적
  - 예: 정원 대비 30~40% 이상
- 신규 모임은 구조적으로 참석자 수가 적을 수 있음

### 가드레일 (신규 모임 보호)
- 신규 모임 생성 후 N일간(예: 14일) 기준 완화 또는 면제
- 모임 규모에 따라 기준 비율 차등 적용
- “참석자 수 적음”이 직접적으로 드러나지 않도록 UI 처리

### 커뮤니케이션
- 신규 유입은 정모 생성 외에도 다양한 경로가 있음을 간접적으로 인지
- 알림을 직접적으로 노출할지 여부는 보류
  - 시스템 메시지 vs. 가이드 문구 vs. 암묵적 구조 유도 검토 필요

---

## 2. 신규 모임 가입자의 정모 참석 허들 완화

### 문제 인식
- 신규 가입자는 기존 멤버 중심 정모에 참여하기 어려움
- 정모 자체가 진입 장벽으로 작용 가능

### 아이디어
- 정모 생성 시 **신규 친화적 속성**을 명시적으로 표현
  - prefix / 라벨 형태
    - 예: `신규 환영`, `초보자 OK`, `장비 필요 없음`
- 카테고리별 맞춤 표현
  - 운동: 초보자 가능
  - 장비/차량 필요 활동: 준비물 불필요 안내

### 기대 효과
- 신규 가입자의 심리적 부담 감소
- 기존 멤버도 정모 성격을 사전에 인지 가능

---

## 3. 신규 가입자 중심 홈 화면 구성

### 아이디어
- 앱 최초 가입자 또는 초기 사용자에게 **홈 화면 섹션 차별화**
- 목적: 신규 사용자 밀도가 높은 환경 제공 → 심리적 안정감

### 예시 섹션
- 신입을 모집하는 모임들이에요
- 신규 가입자가 많은 모임이에요
- 신규 가입자를 환영하는 모임이에요

### 설계 포인트
- “나만 뉴비”라는 인식 제거
- 신규 참석자도 자연스럽게 참여 가능한 분위기 조성
- 기존 홈 구조와의 충돌 여부 검토 필요

---

## 4. 정모 테이블 도입 필요성 검토

### 현황
- DB 상에 정모 테이블 없음
- 정모 슬롯 4개 제한으로 인해:
  - 게시판 활용
  - 카카오톡 등 외부 플랫폼 이동 발생

### 문제
- 정모 생성·관리의 구조적 제약
- 정모 데이터 기반 분석 및 개인화 불가

### 검토 관점
- 정모를 더 쉽게 생성하는 구조
- 정모 로그를 활용한 개인화 가능성
  - 관심사 추정
  - 참여 성향 기반 추천
- 정모가 단발 이벤트가 아닌 **데이터 단위**로 기능할 수 있는지

### 방향성
- 단순 슬롯 확장이 아닌
  - 정모를 1급 엔티티로 다루는 구조 전환 필요성 검토

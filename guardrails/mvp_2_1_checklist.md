# 홈 화면 노출 밀도 가드레일 체크리스트 (MVP 2-1)

목적  
- 1차 MVP 관찰 기간 동안 **전환 가설을 오염시키지 않도록**
- 홈 화면의 **인지 부하와 분석 노이즈를 최소화**한다

---

## A. 변경 가능 / 불가 범위 확인

### 변경 불가 (Fix)
- [ ] 홈 섹션 구조 유지
- [ ] 홈 섹션 순서 유지
- [ ] 더보기 CTA UI 유지
- [ ] 가로 스크롤 방식(UI/제스처) 유지
- [ ] 추천/정렬 로직 변경 없음
- [ ] 이벤트 정의 및 트리거 변경 없음

### 변경 허용
- [ ] 섹션별 **기본 노출 개수** 조정
- [ ] 가로 스크롤 **총 제공 개수 상한** 설정
- [ ] 노출 개수 관련 **예외 처리 규칙** 정의

---

## B. 섹션별 적용 확인

> n * m 형식으로 표시  
> n = 한 화면에서 보여지는 아이템 개수  
> m = 총 스크롤 가능 수

- [ ] 활동이 활발한 모임
    - [ ] 3 * 4
- [ ] 다가오는 정기 모임
    - [ ] 4 * 3
    - [ ] 3번 스크롤하면 자동으로 다음 날짜로 이동함
    - [ ] 자동으로 다음 날짜로 이동했을 시, 더보기 CTA도 해당 날짜에 맞춰서 리스트업
- [ ] 맞춤 모임
    - [ ] 3 * 4
- [ ] 최근 본 모임과 비슷한 모임
    - [ ] 3 * 4
- [ ] 초대받은 모임
    - [ ] 1 * 10
    - [ ] 10번째 아이템 다음 `> 전체보기` 버튼 유지
- [ ] 내 주변에 새로 생겼어요
    - [ ] 3 * 4
- [ ] 찜한 모임에서 정모를 개설했어요
    - [ ] 4 * 3
- [ ] 배우면서 친해져요!
    - [ ] 4 * 3
- [ ] 이런 활동 찾고 있었나요?
    - [ ] 1 * 10
    - [ ] 10번째 아이템 다음 `> 전체보기` 버튼 유지
- [ ] 원하는 모임을 찾지 못했나요?
    - [ ] 기존 UI 및 무한 스크롤 유지

---

## C. 예외 처리 규칙 체크

- [ ] 아이템 수 < 기본 노출 개수  
  → 존재하는 아이템만 노출
- [ ] 아이템 수 = 0  
  → 섹션 유지, 기존 Empty UI 사용
- [ ] 아이템 수 > 가로 총 제공 상한  
  → 상한까지만 전달/렌더
- [ ] **정렬/추천 로직은 그대로 유지**

---

## D. 데이터 수집

홈 노출 밀도 조정과 관련해

- 신규 이벤트 추가 없음
- 기존 행동 이벤트(event_saved / joined / group_joined)로 충분
- 홈 관련 데이터는 성과 측정이 아니라 이상 감지용으로만 사용

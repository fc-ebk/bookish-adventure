# 알림 전면 개편 가드레일 (Final)

본 문서는 1차 MVP(DLV-2026-003) 기간 동안  
알림(Alert / Push)으로 인해 사용자 행동 신호가 왜곡되지 않도록  
허용·금지 기준과 구현 범위를 정의한다.

---

## 1. 기본 원칙

- 알림은 전환을 유도하는 수단이 아니다.
- 알림은 이미 발생한 상태 변화를 전달하는 수단이다.
- 1차 MVP 기간 동안 알림 UX는 실험 대상이 아니다.

---

## 2. Alert / Push 역할

### Alert (앱 내)
- 사용 조건: 앱 foreground
- 역할: 맥락 상기
- 제한:
  - 희소하게 노출
  - 설득 목적 사용 금지
  - 반복 노출 금지

### Push
- 사용 조건: 앱 외부
- 역할: 상태 변화 통지
- 제한:
  - 즉시 행동 가능한 경우만 허용
  - 빈도 제한 필수

---

## 3. 알림 노출 공통 조건

아래 조건을 모두 충족해야 알림을 고려할 수 있다.

1. 실제 상태 변화 발생  
   (자리 발생 / 일정 임박 / 신규 개설)

2. 해당 상태 변화는 사용자에게 손실 가능성을 가진다.

3. 확인 즉시 수행 가능한 행동이 존재한다.

---

## 4. Push OFF 사용자 — 권한 유도 Alert

### 4.1 Alert 성격
- 상태 알림이 아니다.
- 기능 안내가 아니다.
- Push 권한 요청을 위한 사전 맥락 제공(pre-permission)이다.

---

### 4.2 노출 자격 (모두 충족)

- Push OFF 사용자
- 앱 foreground
- 실제 상태 변화 발생
- 해당 상태 변화는 원래 Push 대상 이벤트
- 사용자가 해당 대상과 **최근 연관성**을 가짐  
  (찜 또는 최근 상세 조회 등 최소 1개)
- 빈도 제한 통과

---

### 4.3 노출 흐름

1. 상태 변화 이벤트 수신  
   → UI 노출 없음, 내부 후보로만 저장

2. 노출 기회 발생  
   (앱 foreground 전환 등)

3. **현재 상태 재검증**  
   - 여전히 유효한 상태인지 확인
   - 이미 참여/마감/취소 여부 확인

4. 모든 조건 통과 시  
   → system alert 1회 노출

유효하지 않으면 후보 폐기.

---

### 4.4 명시적 금지 시점

- 찜 직후
- 앱 진입 시
- 홈/탭 진입 시
- 상태 변화가 없는 경우

---

## 5. 빈도 제한

- 사용자 기준: 7일 1회
- 동일 대상 기준: 1회
- dismiss 이후 동일 대상 재노출 금지
- 동일 세션 중복 노출 금지
- lifecycle 전환만으로 재노출 금지

---

## 6. UI 기준

- OS system alert 사용
- custom UI 전환 금지
- 버튼 수·역할 변경 금지
- 애니메이션 및 강조 효과 금지

---

## 7. 타이밍 변경 정책

### 허용
- 무조건 노출되는 Alert 제거
- 의미 없는 Alert 제거
- 빈도 제한 적용

### 금지
- Alert 노출 시점 변경
- 노출 위치 변경
- 묶음·지연 노출 등 UX 재설계

---

## 8. 데이터 정책

### 수집
- Alert / Push 발생 여부

### 제외
- 클릭률 분석
- Push ON 전환율 분석
- 알림 관련 A/B 테스트

---

## 9. 1차 MVP 변경 허용 범위

### 허용
- 잘못 노출되던 Alert 제거
- Alert 노출 조건 명확화
- 빈도 제한 추가

### 금지
- Alert UI 변경
- Alert 설득 강화
- 알림을 통한 전환 증대 시도

---

## 10. 운영 기준

1차 MVP 기간 동안 알림 변경은  
“잘못 뜨는 것을 제거하는 수준”으로만 허용한다.

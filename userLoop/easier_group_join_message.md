# 가입인사 → 채팅 노출 구조 개선안

## 목적

- 무의미한 가입인사(“안녕하세요”, “.”) 감소
- 모임 가입 심리적 허들 완화
- 신규 멤버의 실제 **참여 맥락**을 구성원에게 전달
- 기존 멤버의 문화적 저항 최소화

---

## 1. 문제 정의

### 현재 구조의 한계

1. 자유 텍스트 입력
   - 대부분 형식적 인사로 수렴
2. 정보 가치 부족
   - 왜 참여했는지 알 수 없음
3. 심리적 부담
   - 첫 발화를 강제
4. 의도 불일치
   - 허들 완화 목적과 실제 UX 충돌

---

## 2. 설계 원칙

가입인사는 “인사말”이 아니라  
**입장 알림 + 참여 맥락 공유**로 재정의한다.

- 대화 요청 아님
- 상태 변화 공지
- 개인 발화 아님
- 구조화된 정보 전달

---

## 3. 채팅 노출 방식 개선

### 3.1 기존 방식

```

안녕하세요!

```

---

### 3.2 변경 방식 (권장)

#### 포맷 A. 시스템 알림형

```

[입장 알림]
OOO님이 모임에 참여했어요.
참여 이유: 지금 참여할 수 있는 일정이 있어서

```

#### 포맷 B. 자연어 최소화형

```

OOO님이 참여했어요.
“지금 참여할 수 있는 일정이 있어서”

```

- 말풍선이 아닌 알림 스타일
- 답장을 요구하지 않는 톤
- 채팅이지만 대화로 인식되지 않도록 설계

---

## 4. UI 레벨 처리

- 일반 채팅과 다른 시각 요소 사용
  - 연한 배경
  - 입장/참여 아이콘
  - “모임 알림” 레이블
- 답장 버튼 미노출 또는 비강조

효과:
- 기존 멤버는 반응하지 않아도 어색하지 않음
- 신규 멤버는 말을 걸었다는 부담이 없음

---

## 5. 기존 멤버 설득 전략

### 5.1 기존 멤버의 우려

- 가입인사가 성의 없어 보일 수 있음
- 신규 멤버 정보를 알기 어렵다는 인식
- 모임 분위기가 가벼워질 것이라는 불안

---

### 5.2 설득 논리

형식적인 인사를 없애는 것이 아니라  
**의미 있는 정보로 대체**한다는 점을 명확히 한다.

#### 운영자 공지 예시

```

가입 알림 방식이 바뀌었어요.
형식적인 인사 대신,
이 시점에 함께하게 된 이유를 공유하도록 개선했습니다.

```

---

### 5.3 멤버가 실제로 얻게 되는 변화

기존:
- “안녕하세요” 반복
- 대화로 이어지지 않음

변경 후:
- 참여 계기 확인 가능
- 대화 시작 소재 확보

---

## 6. 사용자 인지부조화 대응

### 문제

선택한 문구와  
채팅에 게시되는 문구가 완전히 동일하지 않을 경우 발생 가능

---

### 대응 원칙

사용자가 “내가 썼다”가 아니라  
“선택한 정보가 공유됐다”고 인식하도록 설계한다.

---

### 6.1 사전 고지

선택 화면 하단 안내

```

선택한 내용은 모임 채팅방에
참여 알림으로 공유됩니다.

```

---

### 6.2 문구 처리 기준

- 과장 금지
- 의미 보존
- 감정 표현 제거
- 1인칭 최소화

나쁜 예:
- 선택: “추천으로 보여서”
- 채팅: “추천받아 꼭 참여하고 싶었습니다”

좋은 예:
- 선택: “추천으로 보여서”
- 채팅: “추천으로 보고 참여했어요”

---

### 6.3 ‘내 말’처럼 보이지 않게 처리

- 시스템 메시지 스타일 유지
- 구조화된 문장 사용
- 따옴표 또는 레이블로 간접화

---

## 7. 구조 변화 요약

| 항목 | 기존 | 변경 |
|---|---|---|
| 의미 | 인사 | 입장 알림 |
| 작성 방식 | 자유 텍스트 | 선택 기반 |
| 채팅 성격 | 대화 시작 | 상태 공유 |
| 정보 가치 | 낮음 | 참여 맥락 제공 |
| 심리적 부담 | 높음 | 낮음 |

---

## 한 줄 요약

가입인사는  
사람의 말을 강제하는 장치가 아니라  
참여 사실과 맥락을 공유하는 정보로 설계해야 한다.
